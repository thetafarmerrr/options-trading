# 期权套利学习计划 v3 — 完整大纲

> 设计原则：计算机做扫描，人做判断。不练记忆力，练判断力和速度。
> 更新：2026-06-22

---

## 整体结构

```
Phase 1 — 市场认知（1周）         练"这是什么东西"
Phase 2 — 波动率基础（2周）       练"期权是贵还是便宜"
Phase 3 — 波动率套利（2周）       练"贵了卖便宜了买"
Phase 4 — 结构套利（2周）         练"同一东西不同价格就是套利"
Phase 5 — 事件驱动（2周）         练"事件前怎么下注"
Phase 6 — 系统整合（持续）        练"什么天气用什么策略"
```

每个 Phase 包含四个组件：
- **核心知识**（读什么、看什么）
- **速度训练**（drill_system.py 对应模块）
- **项目交付**（可运行的脚本）
- **退出考试**（量化验证）

## Phase 1：市场认知 — "这是什么东西"

**目标**：理解期权链结构 + 建立扫描习惯 + 能认出异常
**时间**：1 周（Day 1-7）
**每日时间**：2.5 小时

### Day 1-2：看懂一张期权链

| 时段 | 内容 | 时长 |
|------|------|------|
| 上午 | 拉取沪金 au2608 链面 → 理解每一列的含义 | 60 min |
| | **不是手抄**。对着链面回答：哪个是行权价？哪个是买价？哪个是卖价？虚值区在哪？ | |
| 上午 | 阅读 Natenberg 第 1 章（期权基础） | 30 min |
| 下午 | 拉取 3 个品种（玉米、豆粕、PTA），对比链面差异 | 30 min |
| | 行权价间距一样吗？合约乘数一样吗？谁价差窄？谁最便宜？ | |
| 晚间 | **训练 A — 链面扫异常**（`drill_system.py A`） | 30 min |

### Day 3-4：理解价格关系

| 时段 | 内容 | 时长 |
|------|------|------|
| 上午 | 阅读 Natenberg 第 2-3 章（定价基础、Greeks 入门） | 45 min |
| 上午 | **训练 B — 心算价差**（`drill_system.py B`） | 30 min |
| 下午 | 拉取同品种 2 个不同到期月的链面 | 30 min |
| | 同美元月 Put 贵还是远月贵？为什么？ | |
| 晚间 | **训练 D — Greek 直觉**（`drill_system.py D`） | 30 min |
| | 复习：Delta、Theta、Vega 一句话含义 | 15 min |

### Day 5-6：建立扫描习惯

| 时段 | 内容 | 时长 |
|------|------|------|
| 上午 | 学用 `unified_scanner.py`，理解输出每行含义 | 30 min |
| 上午 | 阅读 Natenberg 第 4 章（波动率） | 45 min |
| 下午 | 扫描全部 10 品种，记录：哪些有信号？价差合理吗？ | 30 min |
| 晚间 | **训练 C — 天气→策略**（`drill_system.py C`） | 30 min |
| | 当天扫描结果和训练 C 场景对照 | 15 min |

### Day 7：Phase 1 退出考试

```
项目交付：
  ✅ 连续 3 天跑 unified_scanner 并记录结果（截图/日志）

速度考试（30 min）：
  训练 A：15 题，找出链面异常         → 通过：正确率 > 80%，均时 < 15s
  训练 B：25 题，心算价差              → 通过：正确率 > 80%，均时 < 6s

书面考试（15 min）：
  1. 看跌期权：行权价越高，价格越__？  → 越贵
  2. 你买入期权，成交价是买价还是卖价？ → 卖价
  3. 价差 = (__ - __) / __ × 100%     → (卖价 - 买价) / 买价
  4. Theta 对买方是__面的，对卖方是__面的 → 负 / 正
  5. 虚值期权：行权价离期货价越__，价格越__ → 远 / 便宜
  6. 吾剑策略核心关注区的行权价特征是？  → 深度虚值，权利金 < 2 元
  7. 扫描器显示"0个可交易"意味着什么？  → 今天不做，正常
  8. 你的 ¥10,000 本金，虚值层单笔上限？→ ¥200
  9. 事件层单笔上限？                   → ¥1,000
  10. 早上扫描后"无信号"，今天该做什么？ → 跑训练，积累数据

通过线：书面 8/10 + 速度双通过
```

## Phase 2：波动率基础 — "期权是贵还是便宜"

**目标**：理解 IV/HV/Greeks → 能判断期权定价水平
**时间**：2 周（Day 8-21）
**每日时间**：2.5 小时

### 知识

| 资源 | 章节 | 完成日 |
|------|------|--------|
| Natenberg《期权波动率与定价》 | 第 4-7 章（波动率、Greeks 深入） | Day 8-14 |
| B站「期权 Greeks 通俗讲解」 | 搜索"期权 Greeks" | Day 10 |
| Sinclair《波动率交易》 | 第 1-2 章（IV 与 RV） | Day 15-18 |
| Tastytrade「Market Measures」 | IV Rank 实操视频（YouTube/B站搬运） | Day 18 |

### 速度训练（每天 30 min）

```
周一/四：训练 A（链面扫异常）— 提高难度，加入 IV 判断
周二/五：训练 B（心算价差）— 增加 OTM/ATM/ITM 混合题
周三/六：训练 C（天气→策略）— 重点：IV 高/低 场景
  周日：训练 D（Greek 直觉）— 加入 Vega 和 IV 相关场景
```

### 项目：`iv_collector.py`

```
功能：每天拉取 5 个品种的 ATM 期权 IV，存入 CSV
列：日期、品种、ATM 行权价、Call IV、Put IV、价差%
验收：连续跑 5 天不报错，输出 CSV 可读
```

### 退出考试

```
书面（20 min）：
  1. IV 和 HV 的区别？
  2. IV Rank 和 IV Percentile 的区别？
  3. 为什么平值期权 Vega 最大？
  4. 为什么临到期 Gamma 最大？
  5. 买入期权时 IV 越高，你的成本越__？→ 高
  6. 卖出期权时 Theta 对你来说是__？→ 收益
  7. IV 均值回归是什么意思？
  8. 当前玉米 IV 高于历史 90% 分位 → 适合买还是卖？→ 卖
  9. 当前豆粕 IV 低于历史 10% 分位 → 适合买还是卖？→ 买
  10. Vega 为正意味着什么？（买方/卖方？）→ 买方

速度：训练 C 15 题，正确率 > 85%，均时 < 8s

项目：iv_collector.py 交付并验证

通过线：书面 8/10 + 速度通过 + 项目交付
```

## Phase 3：波动率套利 — "贵了卖，便宜了买"

**目标**：IV 均值回归 + 偏度 + 期限结构三策略可执行
**时间**：2 周（Day 22-35）

### 知识

| 资源 | 章节 | 完成日 |
|------|------|--------|
| Sinclair《波动率交易》 | 第 3-6 章（IV 预测、偏度、期限结构） | Day 22-28 |
| ivolatility.com | 每天查 2 个品种的 IV 分位数 | 每天 5 min |
| Tastytrade「IV Rank 实战」 | 视频 | Day 25 |
| 自研：回看历史 IV 极端点 | 找沪金 2025 年 IV 最高/最低点，模拟当时怎么做 | Day 29-32 |

### 速度训练（每天 30 min）

```
本周训练 C 权重加倍：IV 高低场景 → 快速判断买卖方向
训练 A 升级：在链面中同时判断异常 + IV 方向
训练 B 保持：价差判断是基本功
```

### 项目：`iv_ranker.py`

```
功能：读取 iv_collector.py 输出的 CSV
      → 计算各品种 IV 在过去 N 天的分位
      → 输出"偏贵/正常/偏便宜"判断
验收：与 ivolatility.com 手工验证偏差 < 10%
```

### 退出考试

```
书面（20 min）：
  10 道 IV 套利场景判断

速度：训练 C 20 题（IV 场景特化），正确率 > 90%，均时 < 5s

实操：给定 5 个品种的当前 IV 数据 → 选出 1 个最适合做买方/卖方的，说明理由

项目：iv_ranker.py 交付

通过线：书面 8/10 + 速度通过 + 实操逻辑正确 + 项目交付
```

## Phase 5：事件驱动 — "事件前怎么下注"

**目标**：事件分析 + 跨式/宽跨式策略可执行
**时间**：2 周（Day 50-63）

### 知识

| 资源 | 章节 | 完成日 |
|------|------|--------|
| Sinclair《波动率交易》 | 第 9-10 章（事件交易） | Day 50-53 |
| Tastytrade「Earnings Plays」 | 视频 | Day 54 |
| 复盘 3 个历史事件 | USDA 报告 ×2 + FOMC ×1 | Day 55-60 |

**复盘模板：**
```
事件：________  日期：________
事件前 3 天 ATM 跨式成本：¥_____（市场定价波动 ___%）
事件后实际跳空幅度：___%
结论：市场高估/低估了波动？
如果当时买跨式：盈亏 ¥_____
```

### 速度训练（每天 30 min）

```
训练 C 权重最大：事件场景 → 策略匹配 → 时限缩短到 4s
```

### 项目：`event_analyzer.py`

```
功能：输入品种 + 事件日期 → 自动拉数据
      → 输出"市场定价波动 vs 历史波动"
      → 给出策略建议
验收：对历史 3 次 USDA 事件，分析结果与事后实际一致
```

### 退出考试

```
案例分析（30 min）：
  给定 2 个未见过的事件场景
  → 计算预期波动 vs 市场定价
  → 选择策略 + 仓位
  → 写"为什么进场"

速度：训练 C 事件特化版，20 题，正确率 > 90%，均时 < 4s

项目：event_analyzer.py 交付

通过线：案例逻辑正确 + 速度通过 + 项目交付
```

## Phase 6：系统整合 — 持续状态

**目标**：多策略轮转 + 自主判断天气 + 实盘执行
**时间**：第 64 天起，持续

### 日常流程

```
08:45  扫描 + 天气判断（5 min）
09:00  根据天气选策略 + drill 对应模块（30 min）
09:30  有信号 → position_check → 下单 / 无信号 → 继续学习或项目迭代
全天   实盘监控（如有持仓）
晚间   复盘 + 数据记录 + 周考（周末）
```

### 项目：`unified_scanner_v3.py`

```
整合 Phase 2-5 全部模块：
  IV 层（iv_ranker）
  + 结构层（parity_scanner）
  + 事件层（event_analyzer）
  → 一键输出"今日做什么"
```

### 持续指标

```
每周：
  跑 unified_scanner 次数 ≥ 5 次
  drill 训练次数 ≥ 4 次
  交易日志至少 1 条（即使是"今日无信号不做"）

每月：
  策略回顾：哪个策略信号最多？哪个最准？
  IV 数据积累 + 分位数更新
  胜率/盈亏比/回撤统计
```

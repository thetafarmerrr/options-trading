# 期权交易学习笔记 — 2026-06-23

## 一、深度虚值链面的真实特征

### bid 跳动是日常，不是异常
- 真实期权链的 bid 不单调——0.50 → 0.00 → 0.50 → 0.00 是深度虚值区的常态
- 原因：每一档的买价来自不同市场参与者，深虚合约经常无人挂单
- P价格（theo）是光滑递增的，但市场只显示 bid/ask

### 三类"异常"，只有一类能做
| 类型 | 特征 | 操作 |
|------|------|------|
| 真倒挂 | 高行权价更便宜 + bid>0 + 价差<50% | 可以做 |
| 假倒挂 | 高行权价更便宜 + bid=0 或价差>100% | 流动性垃圾，忽略 |
| 零买价/价差过宽 | 做市商拒报价 | 碰了就亏 |

## 二、看跌倒挂的正确判断逻辑（已修正）

### 看跌铁律
**行权价越高 → Put 越贵。** 违反此规律 = 倒挂。

```
正常：P664(0.40) < P672(0.48)  ← 低行权价便宜，高行权价贵 ✓
倒挂：P688(3.98) > P696(3.45)  ← 高行权价反而更便宜 ✗
```

### 修正的 Bug
三个文件的 Put 倒挂检测条件全部反了：
- `unified_scanner.py`：`cur.price < nxt.price` → `cur.price > nxt.price`
- `scan_all.py`：同上
- `anomaly_checker.py`：`p > prev[1]` → `p < prev[1]`

修正前把正常价格递增当信号报，修正后只报真正的倒挂。

## 三、训练系统的设计逻辑

### 为什么去掉初级模式
- 真实市场从来不会给你看"干净链"再切换到"脏链"
- 四列显示（买价/卖价/P价格/价差%）本身就是学习工具——P价格列提供"正常递增基准线"
- 在噪音中练识别 = 更接近实盘

### 训练A当前模式
- 10题/轮（--quick 降为5题）
- 链面：0-3个混合异常（真倒挂+假倒挂+零买价+价差宽）
- 任务：只找出可交易的真倒挂
- 正确率目标 >85%

### 训练界面四列的结构
```
行权价 | P买价 | P卖价 | P价格 | 价差%
```
- P买价/P卖价 = 市场真实报价（有跳动）
- P价格 = 光滑趋势线（判断价格递增/递减）
- 价差% = 一键判断流动性（5%能做，50%不能）

- 只在金价单边暴涨+机构踩踏对冲+新行权价不断挂牌时有效
